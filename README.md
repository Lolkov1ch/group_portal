# Group Portal

Портал групи – навчальний Django-проєкт для організації життя студентської групи.  
У проєкті передбачені модулі:

- форум;
- оголошення;
- електронний щоденник;
- навчальні матеріали;
- події та календар;
- опитування та голосування;
- галерея;
- портфоліо студентів;
- система автентифікації та профілі користувачів;
- головна сторінка з оглядом основних розділів.

---

## Технології

- Python 3.10+  
- Django 4+  
- SQLite (за замовчуванням)  
- Bootstrap / власні стилі для фронтенду

---

## Встановлення та запуск проєкту

### 1. Клонування репозиторію

```bash
git clone https://github.com/Lolkov1ch/group_portal.git
cd group_portal
````

### 2. Створення та активація віртуального середовища

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate
```

### 3. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 4. Міграції бази даних

```bash
python manage.py migrate
```

### 5. Створення суперкористувача (адмін)

```bash
python manage.py createsuperuser
```

### 6. Запуск сервера розробки

```bash
python manage.py runserver
```

Далі відкрий у браузері:
`http://127.0.0.1:8000/`

## GitFlow для цього проєкту

Основна робоча гілка репозиторію – **`develop`**.
Пряму роботу в `develop` **заборонено** комітити: кожне завдання виконується в окремій фічевій гілці.

### 1. Іменування гілок

Кожен таск у Trello має свій ID, наприклад:

* `GL-1`, `GL-2` – глобальні/інфраструктурні задачі;
* `UI-1`, `UI-2` – фронтенд/головна сторінка;
* `AUTH-1` – автентифікація;
* `FOR-1`, `FOR-2` – форум;
* `MAT-1` – матеріали;
* `DIA-1` – щоденник;
* `PORT-1` – портфоліо;
* `POL-1` – опитування/голосування;
* `EVT-1` – події.

**Назва гілки = ID таска.**
Наприклад, для завдання `[FOR-2]` гілка має називатись `FOR-2`.

### 2. Робочий цикл для одного таска

1. Переключаємося на актуальний `develop` і оновлюємо його:

   ```bash
   git checkout develop
   git pull origin develop
   ```

2. Створюємо гілку для таска:

   ```bash
   git checkout -b FOR-2
   ```

3. Робимо зміни в коді, локально запускаємо проєкт і перевіряємо, що все працює.

4. Додаємо файли в коміт:

   ```bash
   git add .
   ```

5. Робимо коміт з повідомленням за шаблоном:

   ```bash
   git commit -m "FOR-2 - коротко-що-робиш"
   ```

6. Відправляємо гілку на GitHub:

   ```bash
   git push origin FOR-2
   ```

7. Створюємо **Pull Request** з гілки `FOR-2` → `develop`.

8. У Pull Request обов’язково:

   * у полі **Title** вказати ID та короткий опис, наприклад:
     `FOR-2: список категорій і тем форуму`;
   * у полі **Reviewers** призначити:

     * керівника проєкту (**Доцин Андрій(Lolkov1ch)** або **(Angellllna)**);
   * у полі **Assignees** вибрати **себе**.

9. Після рев’ю та схвалення PR керівник проєкту мерджить його в `develop`.
   Після мерджу фічеву гілку можна видалити на GitHub.

## Правила комітів

* Один коміт – одна логічна зміна (по можливості).
* Обов’язково вказувати ID таска на початку повідомлення:
  `FOR-2 - ...`, `GL-3 - ...`, `AUTH-2 - ...` тощо.
* Не комітити в `develop` напряму.
